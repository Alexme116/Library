Descripcion del programa: El programa se basa en la descripcion proporcionada. Inicialmente hay un menu principal donde se puede acceder a la interfaz de la libreria donde se agrega libros, registran patrones, etc. Luego existe el menu de un usuario normal que sirve para hacer consultas con busqueda y ver todos las librerias. A su vez cuenta con el sistema de patrones donde se pueden pedir prestados libros, consultarlos y regresarlos. Todo esto viene ademas con manejo de errores que aunque como vimos en clase nos siempre es la mejor practica para evitar inputs inesperados, sirve para que el programa no se rompa al momento de colocar inputs que no estan dentro del rango esperado. De igual forma se considera que los libros tienen un numero de copias por lo que al prestar o regresar sumaremosal numero de copias. Por ultimo tenemos que creamos errores en situaciones como cuando no se encuentran libros a pesar de que tampoco son ideales para el "performance" de nuestro programa sirvieron para experimentar con los try catch blocks.
